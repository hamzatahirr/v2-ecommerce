# Multi-Vendor E-Commerce Platform - Technical Report

## Project Overview

This is a comprehensive multi-vendor e-commerce marketplace designed for a BBA student's competition project. The platform enables multiple sellers to list and sell products while providing customers with a unified shopping experience. It features role-based access control, real-time communication, payment integration, and comprehensive admin tools.

## Technical Architecture

### 1. Project Structure

The project follows a monorepo structure with separate frontend and backend applications:

```
v2-ecommerce/
├── client/                 # Next.js 15 Frontend (React 19)
│   ├── app/               # Next.js App Router
│   │   ├── (auth)/        # Authentication pages
│   │   ├── (private)/     # Protected routes
│   │   ├── (public)/      # Public pages
│   │   ├── components/    # Reusable UI components
│   │   ├── store/         # Redux store configuration
│   │   ├── gql/           # GraphQL queries/mutations
│   │   └── types/         # TypeScript definitions
│   └── public/           # Static assets
├── server/                # Express.js Backend (Node.js)
│   ├── src/
│   │   ├── modules/      # Business logic modules
│   │   │   ├── auth/     # Authentication
│   │   │   ├── user/     # User management
│   │   │   ├── product/  # Product management
│   │   │   ├── order/    # Order processing
│   │   │   ├── payment/  # Payment handling
│   │   │   ├── chat/     # Real-time messaging
│   │   │   ├── wallet/   # Seller wallet system
│   │   │   └── admin/    # Admin functions
│   │   ├── infra/        # Infrastructure layer
│   │   │   ├── database/ # Database configuration
│   │   │   ├── cache/    # Redis configuration
│   │   │   ├── payment/  # Payment gateway configs
│   │   │   └── socket/   # WebSocket setup
│   │   ├── shared/       # Shared utilities
│   │   ├── graphql/      # GraphQL schema & resolvers
│   │   └── routes/       # API route definitions
│   └── prisma/           # Database schema & migrations
├── docker-compose.yml    # Local development setup
├── Dockerfile           # Production containerization
└── railway.json        # Railway deployment config
```

### 2. Technology Stack

#### Frontend Technologies
- **Framework**: Next.js 15 with App Router
- **Language**: TypeScript
- **UI Library**: React 19
- **Styling**: Tailwind CSS v4
- **State Management**: Redux Toolkit + RTK Query
- **Data Fetching**: Apollo Client (GraphQL)
- **Real-time Communication**: Socket.io Client
- **Forms**: React Hook Form + Zod validation
- **Charts**: ApexCharts + React-ApexCharts
- **Animations**: Framer Motion
- **Icons**: Lucide React

#### Backend Technologies
- **Runtime**: Node.js 18+
- **Framework**: Express.js
- **Language**: TypeScript
- **Database**: PostgreSQL with Prisma ORM
- **Caching**: Redis (ioRedis)
- **Authentication**: JWT + Express Sessions
- **API**: REST (v1/v2) + GraphQL
- **Real-time**: Socket.io Server
- **File Storage**: Cloudinary
- **Payment Gateways**: JazzCash (Primary), Stripe (Secondary)
- **Email**: Nodemailer with SMTP
- **Documentation**: Swagger/OpenAPI
- **Logging**: Winston
- **Security**: Helmet, CORS, Rate Limiting, XSS Protection

#### Development & Deployment
- **Containerization**: Docker + Docker Compose
- **Version Control**: Git
- **Deployment**: Railway (Cloud Platform)
- **Environment Management**: dotenv
- **Build Tools**: TypeScript Compiler, Next.js Build System
- **Code Quality**: ESLint, Prettier

### 3. Database Schema

The platform uses PostgreSQL with a comprehensive schema supporting multi-vendor operations:

#### Core Entities
- **Users**: Supports multiple roles (USER, ADMIN) with seller verification system
- **Products**: Multi-variant support with attributes, categories, and seller associations
- **Orders**: Complete order lifecycle with status tracking
- **Payments**: JazzCash integration with transaction history
- **Cart**: Session-based and user-persistent shopping cart

#### Advanced Features
- **Seller Management**: Profiles, subscriptions, reviews, and payout systems
- **Wallet System**: Automated earning holds (7-day security period) and withdrawal management
- **Real-time Chat**: Messaging between buyers and sellers
- **Product Variants**: SKU management, inventory tracking, and stock movements
- **Commission System**: Category-based commission rates
- **Analytics**: Sales tracking, reporting, and performance metrics

#### Security & Compliance
- **User Verification**: Student ID card and fee challan verification
- **Domain Restrictions**: Allowed domain management for sellers
- **Audit Logging**: Complete activity tracking
- **Data Integrity**: Foreign key constraints and validation

### 4. API Architecture

- **v1 APIs**: Core functionality

  - User Management: `/api/v1/users/*`

  - Orders: `/api/v1/orders/*`
  - Payments: `/api/v1/payments/*`
  - Chat: `/api/v1/chat/*`

- **Seller notifications**
